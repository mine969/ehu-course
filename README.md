# Ethical Hacking University Course

Ethical Hacking University (EHU) is a complete browser-based cybersecurity curriculum covering the path from core computing fundamentals to professional offensive security, defensive thinking, cloud security, red team operations, vulnerability research, malware analysis, AI security, and career preparation.

The course is published as a static GitHub Pages website and runs from a single `index.html` file with the curriculum embedded for fast, standalone reading.

Live site:

```text
https://mine969.github.io/ehu-course/
```

## What This Course Is

EHU is designed as a structured, self-paced cybersecurity learning path. It is not only a list of topics; it is a guided curriculum with chapters, learning paths, progress tracking, recent reading history, reading controls, search, and practical study guidance.

The curriculum emphasizes both chairs:

- Attacker thinking: how systems fail, how vulnerabilities chain, and how professional testing is performed.
- Defender thinking: how attacks are detected, prevented, contained, reported, and remediated.
- Ethics: all offensive content is framed for authorized labs, legal testing, research, and defensive education.

## What You Will Learn

The curriculum starts from fundamentals and builds toward real-world security work:

- How computers, operating systems, memory, filesystems, permissions, and cryptography work
- Networking from Ethernet, IP, TCP/UDP, DNS, HTTP, HTTPS, TLS, and common protocols
- Linux, shell usage, scripting, process inspection, logs, and system administration basics
- Python, Bash, SQL, regex, APIs, Git, and automation for security work
- Web fundamentals and modern web security concepts
- Web pentesting topics including SQL injection, XSS, SSRF, IDOR, CSRF, JWT, OAuth, file upload attacks, deserialization, request smuggling, and attack chains
- API security across REST and GraphQL
- Network pentesting methodology, scanning, exploitation, privilege escalation, pivoting, and reporting
- Active Directory attacks and defense, including Kerberos, roasting, DCSync, Golden Tickets, ACL abuse, ADCS, and hardening
- Cloud security across AWS, Azure, GCP, Kubernetes, CI/CD, IAM, storage exposure, detection, and incident response
- Bug bounty workflow, recon, triage, reporting, scope discipline, and program strategy
- Red team operations, MITRE ATT&CK, C2 concepts, defense evasion, OPSEC, and purple teaming
- Vulnerability research, fuzzing, reverse engineering, exploit development, and disclosure
- OSINT, social engineering awareness, mobile security, wireless security, IoT, hardware, malware analysis, AI security, and career development

## Suggested Learning Paths

### Complete Beginner

Start with Parts 00 through 05. Build foundations before jumping into exploitation. Focus on Linux, networking, web basics, and Python automation.

Recommended order:

1. Getting Started
2. Foundations
3. Networking
4. OS & Linux
5. Programming
6. Web Fundamentals

### Web Security and Bug Bounty

Use this path if your goal is web application security, PortSwigger-style labs, or bug bounty hunting.

Recommended order:

1. Web Fundamentals
2. Web Pentesting
3. API Security
4. Bug Bounty
5. Vuln Research web topics
6. Reporting and Career chapters

### Network and Internal Pentesting

Recommended order:

1. Networking
2. OS & Linux
3. Programming basics
4. Network Pentesting
5. Active Directory
6. Red Team Ops
7. Attack Chains

## Ethical Use

This course is for lawful cybersecurity education only.

Use the material only in:

- Systems you own
- Labs and CTF environments
- Authorized training platforms
- Written-scope penetration tests
- Bug bounty programs where the behavior is explicitly allowed
- Defensive security research and hardening work

Do not use this material to attack, access, scan, disrupt, or exploit systems without explicit permission. Authorization and scope are part of professional security work.

## Running Locally

Because this is a static site, the simplest option is to open `index.html` in a browser.

For the most browser-compatible local experience, serve the folder with a local web server:

```bash
python -m http.server 8000
```

Then open:

```text
http://localhost:8000/
```

## Deployment

The site deploys automatically to GitHub Pages when changes are pushed to `main`.

Deployment workflow:

```text
.github/workflows/deploy.yml
```

The workflow uses:

- `actions/checkout@v4`
- `actions/upload-pages-artifact@v3`
- `actions/deploy-pages@v4`

## Maintenance Notes

- `index.html` is the deployed application and contains the embedded curriculum data.
- `README.md` documents the course and repository.
- `.github/workflows/deploy.yml` handles GitHub Pages deployment.
- Avoid committing secrets, API keys, private notes, or unrelated generated files.
- After changing `index.html`, run a quick browser check and a static JavaScript syntax check before pushing.
